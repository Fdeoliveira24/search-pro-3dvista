Configuration File Management

How your settings are stored and managed

Settings Storage Information

Settings are stored in your browser's localStorage and persist until you refresh the page, clear browser data or use private browsing.

Use the "Download Config" export options in the toolbar to save your settings to a file for backup or transfer to another device.

Server Configuration

Deploy configuration for all users

Permanent Configuration

Export your settings to a JSON file that can be used for persistent configuration across all browsers and devices.

  1. Click "Download Config" in the toolbar to download the configuration file
  2. Upload the file to /search-pro-config/config/search-config.js on your server
  3. The configuration will automatically load for all users

Import/Export Options

Understanding the different export formats

Download Config

Saves all current settings to a JavaScript file that can be imported later or on another device.

  • Creates a portable settings file
  • Preserves all customizations including colors, layouts, and content settings
  • Can be shared with team members for consistent configuration

Load Config

Import a previously saved configuration file to restore all settings.

  • Supports both .js and .json configuration files
  • Instantly applies all saved settings
  • Perfect for transferring settings between devices

Pro Tips

Best practices for configuration management

Configuration Best Practices

  • Keep a backup of your settings file in a secure location
  • Test configuration changes in a development environment first
  • Use descriptive names when saving multiple configuration variants
  • Document any custom CSS or advanced settings for your team
  • Regularly export your configuration after making significant changes

Troubleshooting

Common issues and solutions

Settings not applying to tour

Make sure the search engine is loaded before applying configuration. Try refreshing the tour page after applying settings.

Configuration file not loading

Verify the file path is correct and the server has proper permissions. Check browser console for any loading errors.

Settings lost after browser restart

This is normal behavior for localStorage. Export your configuration to a file for permanent storage or deploy to server.
